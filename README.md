## Overview

This capstone project demonstrates an end-to-end data pipeline and API system:

1. **Extract** — read raw product review data from `data/reviews.csv`
2. **Transform** — clean the data, compute TextBlob sentiment scores, and calculate per-product rolling average sentiment
3. **Load** — persist results into a SQLite database (`data/reviews_db.sqlite`)
4. **Serve** — expose the processed data through a secured FastAPI service with Swagger UI

---

## Architecture

```
data/reviews.csv
       │
       ▼
┌─────────────────────────────┐
│   ETL Pipeline              │
│   etl_pipeline.py           │
│   ─────────────────────     │
│   Extract → Quality Check   │
│   → Clean → Sentiment NLP   │
│   → Rolling Avg → Load      │
└──────────────┬──────────────┘
               │ SQLite DB
               ▼
┌─────────────────────────────┐
│   FastAPI (app.py)          │
│   ─────────────────────     │
│   GET /health               │
│   GET /api/v1/sentiment/:id │
│   (API Key auth)            │
└─────────────────────────────┘
```

The **Docker build** uses a **multi-stage** strategy:
- **Stage 1 (builder):** runs the ETL pipeline to produce `reviews_db.sqlite` from the CSV
- **Stage 2 (runner):** copies only the compiled database and app code — the heavy CSV never ships in the final image

## Quick Start

### Local Setup (without Docker)

**Prerequisites:** Python 3.10+

```bash
# 1. Clone the repo
git clone https://github.com/Seif-3mk-hassan/Sprints-capstone-project.git
cd Sprints-capstone-project

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set your API key
echo "API_KEY=sprints-secret-key-value" > .env

# 4. Run the ETL pipeline to build the database
python src/etl_pipeline.py

# 5. Start the API server
python src/app.py
```

The API will be available at **http://127.0.0.1:8000**  
Interactive docs (Swagger UI): **http://127.0.0.1:8000/**

---

### Docker (recommended)

```bash
# Build the image (runs ETL automatically during build)
docker build -t reviews-sentiment-api .

# Run the container
docker run -p 8000:8000 -e API_KEY=sprints-secret-key-value reviews-sentiment-api
```

The API will be available at **http://localhost:8000**

## API Reference

### `GET /health`

Check that the server is running and the database is connected. No authentication required.

**Response `200 OK`:**
```json
{
  "status": "ok",
  "database": "connected"
}
```

---

### `GET /api/v1/sentiment/{product_id}`

Retrieve the latest sentiment summary for a given product.

**Authentication:** Pass your API key in the `X-API-Key` request header.

**Path parameter:**

| Parameter    | Type  | Description                      |
|--------------|-------|----------------------------------|
| `product_id` | `int` | Unique identifier for the product |

**Response `200 OK`:**
```json
{
  "product_id": 1001,
  "product_name": "Wireless Headphones",
  "latest_sentiment_score": 4,
  "rolling_average_sentiment": 3.8667
}
```

| Field                       | Type    | Description                                            |
|-----------------------------|---------|--------------------------------------------------------|
| `product_id`                | integer | Product identifier                                     |
| `product_name`              | string  | Name of the product                                    |
| `latest_sentiment_score`    | integer | Star rating from the most recent review                |
| `rolling_average_sentiment` | float   | Rolling average (window=3) of sentiment scores by date |

**Error responses:**

| Status | Meaning                                          |
|--------|--------------------------------------------------|
| `403`  | Missing or invalid `X-API-Key` header            |
| `404`  | No data found for the given `product_id`         |
| `422`  | Validation error (e.g. non-integer `product_id`) |

## ETL Pipeline

Run the pipeline independently at any time:

```bash
python src/etl_pipeline.py
```

**Pipeline stages:**

| Stage                  | Description                                                                   |
|------------------------|-------------------------------------------------------------------------------|
| **Extract**            | Loads `data/reviews.csv` with error handling for missing/corrupt files        |
| **Quality Assessment** | Reports null values, duplicate rows/IDs, and type mismatches                  |
| **Clean**              | Fills nulls, normalizes unicode/whitespace/casing, parses dates, drops dupes  |
| **Sentiment Scoring**  | Computes TextBlob polarity score `[-1.0, 1.0]` for each `review_text`         |
| **Rolling Average**    | Calculates a 3-review rolling average sentiment per product, sorted by date   |
| **Load**               | Writes `reviews` and `product_rolling_sentiment` tables to SQLite             |

## Testing

The integration test suite runs against a live API server. Make sure the server is running first.

```bash
# Start the server (in one terminal)
python src/app.py

# Run tests (in another terminal)
cd src
pytest test_api.py -v
```

**Run against a Docker container:**
```bash
BASE_URL=http://localhost:8000 API_KEY=sprints-secret-key-value pytest src/test_api.py -v
```

**Test coverage:**

| Suite                | What it tests                                                         |
|----------------------|-----------------------------------------------------------------------|
| `TestHealthCheck`    | `/health` returns 200, correct response structure and status values   |
| `TestSentimentEndpoint` | Sentiment data for 3 products, field types, rolling average validity |
| `TestAuthentication` | Missing key → 403, wrong key → 403, correct error detail message     |
| `TestErrorHandling`  | 404 for unknown product, 422 for invalid type, response time < 2s    |
| `TestDocumentation`  | Swagger UI and OpenAPI JSON are accessible                           |
